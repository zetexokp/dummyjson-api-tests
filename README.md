# DummyJSON API Tests

## Описание проекта

Данный проект содержит автоматизированные API-проверки для публичного API DummyJSON.

Цель проекта — проверить основные пользовательские сценарии работы API:

- авторизация пользователя;
- получение данных авторизованного пользователя;
- работа с корзинами;
- проверка негативных сценариев.

В качестве инструментов автоматизации использованы:

- Postman;
- Newman;
- JavaScript для написания API assertions.


---

# Используемые инструменты

В проекте использовались:

- **Postman** — создание коллекции запросов и написание тестов;
- **Newman** — запуск Postman Collection через командную строку;
- **JavaScript (Postman Tests)** — написание проверок ответов API;
- **Node.js + npm** — управление зависимостями и запуск тестов.

# Реализованные проверки

Всего реализовано 10 API-сценариев.


---

## 1. Успешная авторизация

Метод:

```
POST /auth/login
```


Проверяется:

- HTTP статус `200`;
- наличие `accessToken`;
- наличие `id` пользователя.


После успешной авторизации сохраняются переменные окружения:

```
token
userId
```


Они используются в следующих запросах.


---

## 2. Неуспешная авторизация

Метод:

```
POST /auth/login
```


Сценарий:

Отправляется запрос с неверным паролем.


Проверяется:

- HTTP статус `400`;
- наличие сообщения об ошибке.


---

## 3. Получение текущего пользователя с токеном

Метод:

```
GET /auth/me
```


Проверяется:

- HTTP статус `200`;
- наличие данных пользователя:
  - id;
  - username;
  - email.


Используется авторизация:

```
Authorization: Bearer Token
```


---

## 4. Получение текущего пользователя без токена

Метод:

```
GET /auth/me
```


Ожидаемое поведение:

HTTP статус:

```
401 Unauthorized
```


Ответ:

```json
{
   "message": "Access Token is required"
}
```


Проверка выполняется без:

- Authorization header;
- Bearer Token;
- сохраненных cookies.


### Особенность API

При запуске через Newman данный тест может падать.

Причина:

DummyJSON может использовать сохраненные cookies авторизации и возвращать успешный ответ:

```
200 OK
```

вместо:

```
401 Unauthorized
```


Ожидаемое поведение:

```
401 Unauthorized
```


Фактическое поведение:

```
200 OK
```


Для ручного запуска в Postman необходимо:

1. Нажать кнопку **Cookies** возле Send.
2. Выбрать домен:

```
dummyjson.com
```

3. Удалить cookies, связанные с авторизацией.
4. Повторить запрос.

# Работа с корзиной


## 5. Получение корзин пользователя

Метод:

```
GET /carts/user/{userId}
```


Проверяется:

- HTTP статус `200`;
- наличие массива `carts`;
- наличие обязательных полей корзины.


После выполнения сохраняется:

```
cartId
```


---

## 6. Получение корзины по ID

Метод:

```
GET /carts/{cartId}
```


Проверяется:

- HTTP статус `200`;
- наличие объекта корзины;
- наличие товаров;
- соответствие ID корзины.


---

## 7. Создание корзины

Метод:

```
POST /carts/add
```


Проверяется:

- HTTP статус `201`;
- создание объекта корзины;
- наличие товаров;
- корректный `userId`.


---

## 8. Обновление корзины

Метод:

```
PUT /carts/{cartId}
```


Проверяется:

- HTTP статус `200`;
- возврат обновленной корзины;
- изменение количества товара.


---

## 9. Удаление корзины

Метод:

```
DELETE /carts/{cartId}
```


Проверяется:

- HTTP статус `200`;
- наличие:

```json
{
   "isDeleted": true
}
```


Также проверяется:

- корректность ID удаленной корзины;
- наличие даты удаления.


После удаления переменная окружения удаляется:

```
cartId
```


---

# Негативные проверки


## 10. Получение корзины с несуществующим ID

Метод:

```
GET /carts/999999
```


Проверяется:

- HTTP статус `404`;
- наличие сообщения об ошибке.


Ожидаемый ответ:

```json
{
    "message": "Cart with id '999999' not found"
}
```
